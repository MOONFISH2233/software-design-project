# 前端测试平台使用指南

## 📋 概述

本项目提供了一个功能完整的Web前端测试平台，可以一键验证数据服务器的所有API功能。

## 🚀 快速开始

### 1. 启动服务器

```bash
cd d:\学习\软件设计\data-server
python app.py
```

服务器将在 `http://localhost:5000` 启动。

### 2. 访问测试页面

在浏览器中打开以下任一地址：
- **测试仪表盘**: http://localhost:5000/test
- **首页** (自动跳转到测试页面): http://localhost:5000

## ✨ 主要功能

### 🔐 认证功能测试

1. **用户登录 (JWT)**
   - 默认账户: admin / admin123
   - 其他账户: user1, user2, user3 / user123
   - 获取JWT Token用于后续安全接口测试

2. **生成 API Key**
   - 需要先完成登录获取Token
   - 为当前用户生成专属API Key
   - 用于API Key认证方式的数据接收

### 🔒 加密解密功能测试

1. **数据加密**
   - 支持任意JSON格式数据
   - 使用AES-256加密算法
   - 返回Base64编码的加密数据

2. **数据解密**
   - 解密之前加密的数据
   - 验证加密解密的正确性
   - 确保数据完整性

### 📥 数据接收功能测试

1. **普通数据接收** (`/api/receive`)
   - 无需认证
   - 公开接口
   - 适合测试基础功能

2. **JWT认证数据接收** (`/api/receive/secure`)
   - 需要JWT Token
   - 记录操作用户
   - 更安全的数据接收方式

3. **API Key数据接收** (`/api/receive/apikey`)
   - 需要API Key
   - 通过请求头 `X-API-Key` 传递
   - 适合第三方系统集成

### 🌡️ 传感器数据测试

1. **皮肤传感器** (`/api/sensor/skin`)
   - 提交湿度、油脂数据
   - 自动生成时间戳
   - 保存到 `data/skin_sensor/` 目录

2. **环境传感器** (`/api/sensor/environment`)
   - 提交温湿度、光照数据
   - 验证必要字段
   - 保存到 `data/environment/` 目录

3. **设备状态** (`/api/device/status`)
   - 提交设备ID、电量、信号强度
   - 监控设备运行状态
   - 保存到 `data/device/` 目录

### 📊 数据统计功能

- 查看已接收文件总数
- 统计各目录文件数量
- 计算总存储空间占用

## ⚡ 一键式全功能测试

### 使用方法

1. 点击页面底部的 **"🎯 开始一键式全功能测试"** 按钮
2. 系统会自动按顺序执行所有测试
3. 实时显示进度条和每个测试的状态
4. 测试完成后显示汇总结果

### 测试流程

```
1. 健康检查 → 2. 用户登录 → 3. 生成API Key → 
4. 数据加密 → 5. 数据解密 → 6. 普通数据接收 → 
7. JWT数据接收 → 8. API Key数据接收 → 
9. 皮肤传感器 → 10. 环境传感器 → 11. 设备状态 → 
12. 获取统计
```

### 测试结果

测试完成后会显示：
- ✅ **总测试数**: 执行的测试总数
- ✅ **通过**: 成功通过的测试数
- ❌ **失败**: 失败的测试数
- 📈 **成功率**: 通过率的百分比

## 🎨 界面特色

### 视觉反馈

- **状态徽章**: 每个测试都有状态指示
  - 🟡 待测试 (黄色)
  - 🟢 通过 (绿色)
  - 🔴 失败 (红色)

- **结果展示框**: 
  - 绿色背景 = 成功
  - 红色背景 = 失败
  - 显示完整的JSON响应

- **进度条**: 批量测试时显示实时进度

### 交互体验

- 悬停效果: 卡片悬停时有阴影和边框变化
- 按钮禁用: 依赖前置条件的按钮会自动禁用
- 自动启用: 完成登录后自动启用相关功能按钮

## 🔧 配置说明

### 服务器地址

在页面顶部的"服务器地址"输入框中可以修改：
- 本地测试: `http://localhost:5000`
- 远程服务器: `http://your-server-ip:5000`

### 测试数据自定义

每个测试卡片都允许自定义测试数据：
- 登录用户名和密码
- 加密的原始数据
- 传感器数值范围

## 🐛 故障排查

### 问题1: 无法连接服务器

**症状**: 所有测试都显示"连接失败"

**解决方案**:
1. 确认服务器已启动 (`python app.py`)
2. 检查服务器地址是否正确
3. 确认防火墙未阻止5000端口

### 问题2: 认证失败

**症状**: 登录测试失败

**解决方案**:
1. 确认使用正确的用户名密码
2. 默认账户: admin/admin123
3. 检查服务器日志查看具体错误

### 问题3: 某些按钮一直是禁用状态

**原因**: 这些功能依赖前置条件

**解决方案**:
- "生成API Key" 需要先完成登录
- "测试解密" 需要先完成加密
- "JWT数据接收" 需要先登录
- "API Key数据接收" 需要先生成API Key
