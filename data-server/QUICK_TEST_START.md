## 快速开始 (3步完成)

### Windows 用户

```bash
# 方法1: 双击运行
start_test_platform.bat

# 方法2: 命令行运行
.\start_test_platform.bat
```

### Linux/Mac 用户

```bash
# 赋予执行权限

# 运行
./start_test_platform.sh
```


## 使用流程

### 方式1: 一键式全自动测试 ⚡

1. 打开测试页面
2. 点击底部的 **"🎯 开始一键式全功能测试"** 按钮
3. 等待自动完成所有12项测试
4. 查看测试结果汇总

**测试内容包括**:
- ✅ 健康检查
- ✅ 用户登录 (JWT)
- ✅ 生成API Key
- ✅ 数据加密/解密
- ✅ 三种数据接收方式
- ✅ 三种传感器接口
- ✅ 数据统计

### 方式2: 手动逐项测试 🎮

每个功能模块都有独立的测试卡片：

1. **认证功能** 🔐
   - 输入用户名密码 → 点击"测试登录"
   - 获取Token后 → 点击"生成API Key"

2. **加密解密** 🔒
   - 编辑测试数据 → 点击"测试加密"
   - 获取加密数据后 → 点击"测试解密"

3. **数据接收** 📥
   - 普通接收：直接点击测试
   - JWT接收：先登录后测试
   - API Key接收：先生成Key后测试

4. **传感器数据** 🌡️
   - 皮肤传感器、环境传感器、设备状态
   - 各点击对应按钮即可测试

5. **数据统计** 📊
   - 点击"获取统计"查看文件统计信息

## 常见问题

### Q1: 为什么有些按钮是灰色的？
A: 这些功能需要先完成前置测试。例如：
- "生成API Key" 需要先登录
- "测试解密" 需要先加密
- "安全数据接收" 需要先登录

### Q2: 如何测试远程服务器？
A: 在页面顶部的"服务器地址"输入框中修改为：
```
http://your-server-ip:5000
```

### Q3: 测试数据可以自定义吗？
A: 可以！大部分测试都支持自定义：
- 登录用户名和密码
- 加密的原始数据
- 其他测试使用随机生成的数据

### Q4: 批量测试失败了怎么办？
A: 
1. 查看失败测试的结果框，了解具体错误
2. 检查服务器是否正常运行
3. 确认网络连接正常
4. 可以尝试单独测试失败的接口
