## 答辩问题准备

### 常见问题及标准答案

#### Q1: 为什么选择Flask而不是Django？

**标准答案**:
> "Flask轻量灵活，适合快速开发RESTful API。配合Gunicorn可实现高并发，满足本项目需求。Django更适合大型Web应用，内置ORM、Admin等功能，但对于纯API服务来说过于重量级。我们的项目追求简洁高效，Flask是最佳选择。"

#### Q2: Redis Streams和传统消息队列有什么区别？

**标准答案**:
> "Redis Streams是Redis 5.0新增功能，优势是轻量、无需额外部署、学习成本低，适合中小规模场景。我们已有Redis基础设施，复用降低成本。RabbitMQ/Kafka更适合企业级应用，支持复杂路由和死信队列，但部署和维护成本高。如果未来业务扩展到百万级QPS，我们会考虑迁移到Kafka。"

#### Q3: 如何保证数据不丢失？

**标准答案**:
> "我们采用三重保障：第一，同步写入JSON文件，数据立即落盘；第二，Redis Stream配置持久化，即使重启也不会丢失；第三，备份机制，定期将数据备份到backup1-4键。即使服务崩溃，数据也能恢复。生产环境可以增加数据库双写和异地备份。"

#### Q4: 安全性如何评估？

**标准答案**:
> "我们实现了四层防护：认证层（JWT/API Key双重认证）、传输层（AES-256加密）、应用层（速率限制防DDoS）、审计层（完整日志记录）。已通过无效Token、过期Token、超频请求等安全测试。生产环境建议增加HTTPS、IP白名单和WAF防火墙。"

#### Q5: 性能瓶颈在哪里？如何优化？

**标准答案**:
> "当前瓶颈在磁盘I/O，因为每条数据都同步写入JSON文件。优化方案包括：批量写入减少I/O次数、使用SSD硬盘提升读写速度、引入数据库（MySQL/PostgreSQL）替代文件存储、异步I/O进一步解耦。理论上优化后可支撑10000+ QPS，当前200+ QPS已满足需求。"

#### Q6: 如果要多机部署怎么办？

**标准答案**:
> "架构天然支持水平扩展：第一，Nginx负载均衡多台Flask服务器；第二，Redis主从复制或集群模式；第三，共享存储（NFS）或分布式文件系统保证数据一致性；第四，数据库读写分离。当前单节点已稳定运行，如需扩展只需增加服务器并配置负载均衡即可。"

## 应急处理方案

### 问题1: SSH连接失败

**症状**: `ssh root@47.103.108.47` 超时或拒绝连接

**解决方案**:
1. 检查网络连接：`ping 47.103.108.47`
2. 检查阿里云安全组：确认5000端口已开放
3. 使用备用截图进行演示
4. 联系老师说明网络问题，请求稍后补演示

### 问题2: Gunicorn服务未运行

**症状**: `ps aux | grep gunicorn` 无输出

**解决方案**:
```bash
# 重启服务
cd /root/data-server
nohup gunicorn -c config/gunicorn_config.py app:app > logs/server.log 2>&1 &

# 验证服务
sleep 3
curl http://localhost:5000/api/health
```

### 问题3: Redis服务未运行

**症状**: `redis-cli ping` 返回错误

**解决方案**:
```bash
# 启动Redis
systemctl start redis

# 验证
redis-cli ping
# 应输出: PONG
```

### 问题4: 演示命令报错

**常见错误**: `ModuleNotFoundError: No module named 'xxx'`

**解决方案**:
```bash
# 安装缺失的依赖
pip3 install flask-limiter cryptography redis

# 验证语法
python3 -m py_compile app.py
```

### 问题5: 数据文件未生成

**症状**: 发送数据后data目录无新文件

**解决方案**:
1. 检查服务端日志：`tail -f /root/data-server/logs/server.log`
2. 检查目录权限：`ls -ld /root/data-server/data/`
3. 手动创建目录：`mkdir -p /root/data-server/data/skin`

### 问题6: PPT无法打开

**解决方案**:
1. 使用WPS Office打开（兼容性更好）
2. 转换为PDF备用：`libreoffice --convert-to pdf 第二阶段验收汇报.pptx`
3. 使用在线预览：上传到OneDrive或Google Drive
