## ❌ 常见错误及修正

### 错误1: 缺少Content-Type导致登录失败
```bash
# ❌ 错误
curl -X POST http://localhost:5000/api/auth/login \
  -d '{"username":"admin","password":"admin123"}'
# 输出: {"error":"缺少用户名或密码"}

# ✅ 正确
curl -X POST http://localhost:5000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'
```

### 错误2: 数据路径不存在
```bash
# ❌ 错误（假设存在子目录）
ls /root/data-server/data/skin/

# ✅ 正确（文件直接在data目录下）
ls /root/data-server/data/*.json
```

## 💡 答辩问题速查

**Q1: 为什么选择Flask？**  
A: 轻量灵活，适合RESTful API，配合Gunicorn可高并发。

**Q2: Redis Streams优势？**  
A: 轻量、无需额外部署、学习成本低，已有Redis基础设施。

**Q3: 如何保证数据不丢失？**  
A: 同步写入JSON + Redis持久化 + 备份机制。

**Q4: 安全性如何？**  
A: 四层防护：认证层 + 传输层 + 应用层 + 审计层。

**Q5: 性能瓶颈？**  
A: 磁盘I/O，可通过数据库、SSD、异步I/O优化。

**Q6: 如何多机部署？**  
A: Nginx负载均衡 + Redis集群 + 共享存储/NFS。
