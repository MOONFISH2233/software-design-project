📚 学习要点

1. 消息队列 (MQ) 核心概念

概念 说明 应用场景
Producer 消息生产者 模拟器生成数据
Consumer 消息消费者 接收、验证、写入模块
Queue/Stream 消息队列/流 Redis Stream
Consumer Group 消费者组 receiver_group, validator_group
ACK 消息确认 确保消息不丢失

2. Redis Stream 关键命令

# 添加消息到流 XADD sensor:raw * sensor_type skin data "{...}" # 消费者组读取消息 XREADGROUP GROUP receiver_group receiver_1 COUNT 1 STREAMS sensor:raw > # 确认消息已处理 XACK sensor:raw receiver_group # 查看流长度 XLEN sensor:raw # 查看 pending 消息 XPENDING sensor:raw receiver_group

3. 模块化设计原则

  • 单一职责：每个模块只做一件事
  • 松耦合：模块间通过 MQ 通信
  • 可替换：可以独立替换任何一个模块
  • 可扩展：支持启动多个相同功能的模块

4. 容错机制

重传策略

  • 最大重试次数：3 次
  • 重试间隔：30 秒（指数退避）
  • 超时检测：10 秒
  • 失败消息记录到专门的文件
