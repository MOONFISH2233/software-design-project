# 客户端加密
from cryptography.fernet import Fernet
cipher = Fernet(encryption_key)

encoded_data = base64.b64encode(encrypted_data).decode()

### ✅ (7) 搭建消息队列，实现部分类型的数据通过消息队列上报到服务器侧

**完成状态**: ✅ 已完成  
**技术选型**: Redis Streams

**架构设计**:
```
模拟器(simulator_mq.py)
    ↓ publish()
Redis Streams
    ├─ sensor:raw (原始数据)
    ├─ sensor:validated (验证后)
    ├─ sensor:write (待写入)
    └─ sensor:logs (日志)
    ↓ consume()
后台消费者
    ↓
JSON文件存储
```

**验证方法**:
```bash
# 1. 检查Redis服务
redis-cli ping
# 输出: PONG

# 2. 查看Streams
redis-cli keys 'sensor:*'
# 输出: sensor:raw, sensor:validated, sensor:write, sensor:logs

# 3. 运行模拟器
cd /root/data-server
python3 simulator_mq.py

# 观察输出:
# 🟢 成功发布 : 156 条
# 💧 [发布成功] MQ接收: skin
# ☀️ [发布成功] MQ接收: environment

# 4. 检查数据文件增长
watch -n 1 'ls /root/data-server/data/skin/ | wc -l'
```

**关键代码** (`mq_utils.py`):
```python
class RedisMQ:
    def publish(self, stream_name, message):
        """发布消息到Stream"""
        self.redis_client.xadd(stream_name, message)
    
    def consume(self, stream_name, consumer_group, consumer_name):
        """消费消息（支持消费者组）"""
        messages = self.redis_client.xreadgroup(
            groupname=consumer_group,
            consumername=consumer_name,
            streams={stream_name: '>'},
            count=10
        )
        return messages
```

**实际结果**:
- ✅ Redis服务正常运行
- ✅ 4个Stream创建成功
- ✅ 模拟器持续发布数据
- ✅ 数据自动保存到文件

## 💡 答辩问题准备

### Q1: 为什么选择Flask而不是Django？
**A**: Flask轻量灵活，适合快速开发RESTful API。配合Gunicorn可实现高并发。若需要ORM、Admin等复杂功能，可选择Django。

### Q2: Redis Streams相比RabbitMQ有什么优势？
**A**: 轻量级、无需额外部署、学习成本低，适合中小规模。本项目已有Redis基础设施，复用降低成本。企业级场景可迁移到RabbitMQ。

### Q3: 如何保证数据可靠性？
**A**: 三重保障：①同步写入JSON文件；②Redis Stream持久化；③备份机制。即使服务崩溃，数据也不会丢失。

### Q4: 安全性如何评估？
**A**: 四层防护：认证层（JWT/API Key）、传输层（AES加密）、应用层（速率限制）、审计层（日志）。生产环境建议增加HTTPS。

### Q5: 性能瓶颈在哪里？如何优化？
**A**: 当前瓶颈在磁盘I/O。优化方案：批量写入、数据库替代文件、SSD硬盘、异步I/O。理论可支撑10000+ QPS。

### Q6: 如果要多机部署怎么办？
**A**: Nginx负载均衡 + Redis集群 + 共享存储/NFS + 数据库读写分离。架构支持水平扩展。
