# 更新日志 (CHANGELOG)

## [2.0.0] - 2026-03-27

### 🚀 重大性能升级

#### 新增功能
- ✅ **异步日志处理器** (`AsyncLogHandler`)
  - 使用独立线程处理日志 I/O，不阻塞主请求线程
  - 队列容量：1000 条日志记录
  - 队列满时自动丢弃低优先级日志（DEBUG/INFO）
  - 保证 ERROR/CRITICAL 级别日志不丢失

- ✅ **日志采样机制** (`SamplingFilter`)
  - 可配置采样率（默认 10%）
  - INFO 和 DEBUG 级别按概率记录
  - WARNING/ERROR/CRITICAL 全量记录
  - 大幅减少高并发下的日志输出量

- ✅ **结构化 JSON 日志**
  - 所有日志以 JSON 格式输出
  - 包含完整上下文信息（时间戳、线程、模块、函数、行号）
  - 便于 ELK、Graylog 等日志分析系统处理
  - 支持异常堆栈追踪

- ✅ **线程安全统计** (`RequestStats`)
  - 使用锁机制保证线程安全
  - 实时统计请求总数
  - 每分钟自动记录 QPS 估算
  - 无需外部数据库即可监控流量

- ✅ **慢请求检测**
  - 自动检测耗时超过 100ms 的请求
  - 单独记录 WARNING 级别日志
  - 帮助快速定位性能瓶颈

- ✅ **错误专用日志通道**
  - ERROR 级别日志单独保存到 `error_YYYYMMDD.log`
  - 不与普通日志混合
  - 便于问题排查和告警

#### 优化改进
- 📈 **日志轮转增强**
  - 普通日志：最大 50MB，保留 10 个备份
  - 错误日志：最大 20MB，保留 15 个备份
  - 避免单文件过大影响读写性能

- 🎯 **控制台输出优化**
  - 仅显示 WARNING 及以上级别
  - 减少控制台刷屏对性能的影响
  - 保持传统格式便于人工阅读

- ⚡ **请求处理优化**
  - 添加请求耗时监控
  - 响应中返回处理时间（`process_time_ms`）
  - 便于客户端监控服务端性能

- 🔧 **生产环境适配**
  - 默认关闭 debug 模式
  - 启用多线程模式（`threaded=True`）
  - 添加 Gunicorn 启动示例注释

#### API 变更
- **新增接口**: `/api/stats`
  - 获取服务器统计信息
  - 显示总请求数、版本、特性列表
  
- **健康检查增强**: `/api/health`
  - 新增 `total_requests` 字段
  - 新增 `uptime` 状态标识

#### 技术栈
- Python 标准库实现（无额外依赖）
- 兼容 Flask 内置开发服务器
- 支持 Gunicorn、uWSGI 等生产级 WSGI 服务器

## 🔧 部署建议

### 开发环境
```bash
python app.py
```

### 生产环境（推荐 Gunicorn）
```bash
# 4 个工作进程，每个进程处理并发请求
gunicorn -w 4 -b 0.0.0.0:5000 --threads=4 app:app

# 高性能模式（根据 CPU 核心数调整）
gunicorn -w $(nproc) -b 0.0.0.0:5000 --threads=8 --max-requests 1000 app:app
```

### 日志配置调整
如需修改采样率，编辑 `app.py`:
```python
sampling_filter = SamplingFilter(sample_rate=0.1)  # 改为需要的值（0.0-1.0）
```

如需修改异步队列大小:
```python
async_handler = AsyncLogHandler(file_handler, queue_size=1000)  # 调整队列容量
```

## ⚠️ 注意事项

1. **日志采样率设置**
   - 开发环境：建议 1.0（100% 记录）
   - 生产环境：建议 0.1（10% 记录）或更低
   - 超高并发：建议 0.01（1% 记录）

2. **队列溢出处理**
   - 队列满时自动丢弃 DEBUG/INFO 日志
   - ERROR/CRITICAL 会等待队列有空位
   - 不会因日志问题导致服务崩溃

3. **日志文件格式**
   - 默认使用 JSON 格式
   - 如需改回传统格式，可切换 formatter

4. **资源占用**
   - 每个异步处理器占用 1 个后台线程
   - 队列占用内存约几 MB（取决于队列大小）
